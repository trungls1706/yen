# 📘 Next.js – Full 50 Câu Hỏi Phỏng Vấn

---

# I. NEXT.JS FUNDAMENTALS

## 1. Next.js là gì? Tính năng chính?

Next.js là một React framework hỗ trợ:

- SSR (Server-Side Rendering)
- SSG (Static Site Generation)
- ISR (Incremental Static Regeneration)
- File-based routing
- API Routes
- Image optimization
- Middleware
- SEO tốt hơn

---

## 2. Next.js khác gì Create React App?

CRA:
- Chỉ CSR
- Không có API route
- SEO kém

Next.js:
- SSR, SSG, ISR
- API route
- Routing tự động
- SEO tốt

---

## 3. Tạo project Next.js bằng lệnh gì?

```bash
npx create-next-app@latest
```

---

## 4. Next.js xử lý SSR như thế nào?

Với Pages Router:
- Dùng `getServerSideProps`

Với App Router:
- Server Component mặc định chạy trên server

Mỗi request sẽ render lại HTML.

---

## 5. Mục đích thư mục pages?

Trong Pages Router:
- Mỗi file tương ứng 1 route
- Hỗ trợ getStaticProps, getServerSideProps

(App Router thay bằng thư mục app)

---

## 6. Tạo page render server mỗi request?

Pages Router:

```tsx
export async function getServerSideProps() {
  return { props: {} }
}
```

App Router:

```tsx
export const dynamic = "force-dynamic"
```

---

## 7. Next.js hỗ trợ file extension nào?

- .js
- .jsx
- .ts
- .tsx

---

## 8. Environment variables hoạt động thế nào?

File `.env.local`

Biến public phải có prefix:

```
NEXT_PUBLIC_API_URL=
```

Biến không prefix chỉ dùng được ở server.

---

## 9. Automatic Static Optimization là gì?

Nếu page không có:
- getServerSideProps
- getInitialProps

Next.js sẽ tự build static HTML.

# II. PAGES & ROUTING

## 10. File-based routing hoạt động ra sao?

app/page.tsx → /
app/about/page.tsx → /about

---

## 11. Dynamic routes tạo thế nào?

```tsx
app/blog/[id]/page.tsx
```

---

## 12. Truy cập URL params?

```tsx
export default function Page({ params }) {
  return <div>{params.id}</div>
}
```

---

## 13. Link component dùng để làm gì?

```tsx
import Link from "next/link"
```

- Điều hướng client-side
- Prefetch tự động
- Không reload page

---

## 14. Catch-all routes là gì?

```tsx
app/docs/[...slug]/page.tsx
```

URL: /docs/a/b/c  
slug = ["a","b","c"]

---

# III. DATA FETCHING

## 15. getStaticProps là gì?

- Chạy lúc build
- Tạo static HTML
- Phù hợp blog

---

## 16. getServerSideProps là gì?

- Chạy mỗi request
- Dùng cho dashboard, auth

---

## 17. ISR triển khai thế nào?

```tsx
fetch(url, { next: { revalidate: 60 } })
```

Rebuild sau 60 giây.

---

## 18. Fetch client-side thế nào?

```tsx
"use client"
useEffect(() => {
  fetch(...)
}, [])
```

---

## 19. fallback trong getStaticPaths là gì?

- false → 404 nếu không tồn tại
- true → render tạm, build background
- blocking → chờ build xong rồi trả về

---

# IV. API ROUTES

## 20. Tạo API route thế nào?

app/api/users/route.ts

```tsx
export async function GET() {
  return Response.json({ ok: true })
}
```

---

## 21. Dùng middleware trong API được không?

Có. Dùng middleware.ts để:

- Check auth
- Redirect
- Validate role

---

## 22. Query params xử lý thế nào?

```tsx
export async function GET(req: Request) {
  const { searchParams } = new URL(req.url)
  const id = searchParams.get("id")
}
```
